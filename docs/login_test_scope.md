# ログインE2Eテスト観点（仮）

## 前提（仮）
- 対象URLは未確定のため、テスト内では仮のURL（例: `https://example.com/login`）を使用する。  
- テスト対象はログイン画面の基本フローのみ（多要素認証やパスワード再設定は対象外）。  
- セレクタは可能なら `role/label/testid` を優先し、無い場合は最小のCSSを仮置きする。  

## テスト観点（最小）
### 正常系
- 正しいID/PWでログインでき、ログイン後の画面（例: ダッシュボード）に遷移する。  

### 異常系（最大3）
- ID未入力でログインを試みると、必須エラーが表示される。  
- PW未入力でログインを試みると、必須エラーが表示される。  
- 誤ったID/PWでログインを試みると、認証エラーが表示される。
